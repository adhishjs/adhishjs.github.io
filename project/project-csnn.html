Overview

This project implements a 2 mm × 2 mm ASIC in SCL 180nm that combines the event‑driven efficiency of Spiking Neural Networks (SNNs) with the feature‑extraction power of convolutional layers. The chip receives serial spike events, applies a 1D temporal convolution, and updates 16 parallel Leaky Integrate‑and‑Fire (LIF) neurons — all in real time.

Designed for edge AI, it handles time‑series spike streams with ultra‑low latency, making it ideal for wearable sensing, gesture recognition, and biomedical signal classification.

How it works

  • 1 Spike reception & shift register

    Serial spike input is packed into 16‑bit vectors over 16 cycles. Three consecutive vectors are stored in a shift register, forming a 3×16 temporal window.

  • 2 Parallel weighted accumulation

    For each of the 16 output channels, 48 weights (3×16) are conditionally accumulated based on spike presence. All channels compute in parallel using a combinational array.

  • 3 LIF neuron update & thresholding

    Each channel updates its independent 16‑bit membrane potential. If (membrane + accumulation) ≥ threshold, a spike is fired and the membrane resets; otherwise it accumulates.

  • 4 Configuration interface

    Runtime programming of all 768 weights and the global threshold via a synchronous interface. Configuration is prioritised to avoid contention with spike reception.

  • 5 Output handshake

    The done signal is asserted for one cycle, and the 16‑bit spikes_out vector is valid. The module then returns to receive mode for the next timestamp.

Applications

PCG classification

Real‑time detection of normal vs. abnormal heart sounds from phonocardiogram spike trains.

Gesture recognition

Processes event‑based sensor (DVS) data using temporal convolution for dynamic gestures.

Keyword spotting

Audio spike streams from analog signals; the temporal kernel captures phoneme‑level features.

Wearable & IoT

Battery‑powered continuous sensing without cloud offload — always‑on edge AI.

The design excels in time‑series spike classification where input arrives as a binary event stream and latency is critical.
